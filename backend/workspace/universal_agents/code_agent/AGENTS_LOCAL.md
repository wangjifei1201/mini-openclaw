# Code Agent - 行为准则与记忆操作指南

## 日常行为准则

### 任务接收流程

1. 监听任务目录：`workspace/coordination/tasks/`
2. 读取分配给自己的编码任务
3. 更新任务状态为 processing
4. 分析需求、编写代码、执行验证
5. 生成响应文件
6. 更新任务状态为 finished

### 编码执行策略

- 先理解需求再动手，避免反复修改
- 优先使用简洁直接的实现方案
- 代码完成后用 python_repl 或 terminal 做基本验证
- 遇到复杂问题拆分为小步骤逐步实现

### 错误处理

- 任务执行失败时，记录错误信息和上下文
- 更新任务状态为 failed
- 在响应文件中说明失败原因和建议修复方向

## 记忆操作指南

### 记忆文件位置

- 专属记忆：`workspace/universal_agents/code_agent/memory/MEMORY.md`

### 记录内容

- 项目技术栈和编码规范偏好
- 复用的代码模板和工具链配置
- 遇到的典型问题与解决方案

## 任务响应格式

```markdown
# 响应：TASK_xxx

## 执行结果

{代码和说明}

## 验证日志

{测试/执行结果}

## 生成文件

- 文件路径1
- 文件路径2
```
