## 设计思路

**不要创建无聊的幻灯片。** 白色背景上的普通项目符号不会给任何人留下深刻印象。为每张幻灯片考虑此列表中的想法。

### 开始前

- **选择大胆、与内容相关的配色方案**：调色方案应该感觉为这个主题专门设计。如果将您的颜色换成完全不同的演示文稿仍然\"有效\"，那么您的选择还不够具体。
- **主导而非平等**：一种颜色应该占主导（60-70% 视觉权重），1-2 种辅助色调和一种鲜明的强调色。永远不要给所有颜色同等权重。
- **深色/浅色对比**：标题和结尾幻灯片用深色背景，内容用浅色（\"三明治\"结构）。或者全程使用深色以获得高端感。
- **坚持视觉主题**：选择一个独特的元素并重复它——圆角图片框、彩色圆圈中的图标、粗单线边框。在每张幻灯片中贯穿使用。

### 配色方案

选择与您的主题匹配的颜色——不要默认使用通用蓝色。使用这些调色板作为灵感：

| 主题 | 主色 | 辅助色 | 强调色 |
|-------|---------|-----------|--------|
| **午夜高管** | `1E2761`（海军蓝） | `CADCFC`（冰蓝） | `FFFFFF`（白色） |
| **森林与苔藓** | `2C5F2D`（森林绿） | `97BC62`（苔藓绿） | `F5F5F5`（奶油色） |
| **珊瑚活力** | `F96167`（珊瑚色） | `F9E795`（金色） | `2F3C7E`（海军蓝） |
| **暖陶土** | `B85042`（陶土色） | `E7E8D1`（沙色） | `A7BEAE`（鼠尾草色） |
| **海洋渐变** | `065A82`（深蓝） | `1C7293`（青绿色） | `21295C`（午夜蓝） |
| **炭灰极简** | `36454F`（炭灰色） | `F2F2F2`（米白色） | `212121`（黑色） |
| **青绿信任** | `028090`（青绿色） | `00A896`（海沫绿） | `02C39A`（薄荷绿） |
| **浆果与奶油** | `6D2E46`（浆果色） | `A26769`（暗玫瑰色） | `ECE2D0`（奶油色） |
| **鼠尾草宁静** | `84B59F`（鼠尾草色） | `69A297`（桉树色） | `50808E`（石板色） |
| **樱桃大胆** | `990011`（樱桃色） | `FCF6F5`（米白色） | `2F3C7E`（海军蓝） |

### 每张幻灯片

**每张幻灯片都需要一个视觉元素**——图片、图表、图标或形状。纯文本幻灯片容易被遗忘。

**布局选项：**
- 两列（左侧文本，右侧插图）
- 图标 + 文本行（彩色圆圈中的图标，粗体标题，下方描述）
- 2x2 或 2x3 网格（一侧图片，另一侧内容块网格）
- 半出血图片（完整的左侧或右侧）与内容叠加

**数据显示：**
- 大数字标注（大号数字 60-72pt，下方小标签）
- 比较列（之前/之后、优点/缺点、并排选项）
- 时间线或流程（编号步骤、箭头）

**视觉润色：**
- 章节标题旁边的小彩色圆圈中的图标
- 关键统计数据或标语的斜体强调文本

### 排版

**选择一个有趣的字体搭配**——不要默认使用 Arial。选择一个有个性的标题字体，并将其与干净正文字体搭配。

| 标题字体 | 正文字体 |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 大小 |
|---------|------|
| 幻灯片标题 | 36-44pt 粗体 |
| 章节标题 | 20-24pt 粗体 |
| 正文 | 14-16pt |
| 说明文字 | 10-12pt 弱化 |

### 间距

- 最小 0.5" 边距
- 内容块之间 0.3-0.5"
- 留出呼吸空间——不要填满每一寸

### 避免（常见错误）

- **不要重复相同的布局**——在幻灯片中变换列、卡片和标注
- **不要将正文居中**——左对齐段落和列表；只居中标题
- **不要在大小对比上吝啬**——标题需要 36pt+ 才能从 14-16pt 正文中脱颖而出
- **不要默认使用蓝色**——选择与特定主题匹配的颜色
- **不要随机混合间距**——选择 0.3" 或 0.5" 间隙并始终使用
- **不要只样式化一张幻灯片而让其余保持普通**——全程完全投入或保持简单
- **不要创建纯文本幻灯片**——添加图片、图标、图表或视觉元素；避免纯标题 + 项目符号
- **不要忘记文本框内边距**——当将线条或形状与文本边缘对齐时，在文本框上设置 `margin: 0` 或偏移形状以考虑内边距
- **不要使用低对比度元素**——图标和文本都需要与背景形成强烈对比；避免浅色背景上的浅色文本或深色背景上的深色文本
- **切勿在标题下使用强调线**——这些是 AI 生成幻灯片的标志；使用空白或背景色代替

## QA（必需）

**假设存在问题。您的工作是找到它们。**

您的第一次渲染几乎从不正确。将 QA 视为错误搜寻，而不是确认步骤。如果第一次检查就找到零问题，说明您看得不够仔细。

### 内容 QA

```bash
python -m markitdown output.pptx
```

检查缺失内容、错别字、错误顺序。

**使用模板时，检查遗留的占位符文本：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 返回结果，在声明成功之前修复它们。

### 视觉 QA

**使用子代理**——即使只有 2-3 张幻灯片。您一直盯着代码，会看到您期望的内容，而不是实际存在的内容。子代理有新鲜的眼光。

将幻灯片转换为图片（参见 [转换为图片](#converting-to-images)），然后使用以下提示：

```
直观检查这些幻灯片。假设存在问题——找到它们。

查找：
- 重叠元素（文本穿过形状、线条穿过文字、堆叠元素）
- 文本溢出或在边缘/框边界处被截断
- 为单行文本定位的装饰线但标题换行到两行
- 与上方内容碰撞的来源引用或页脚
- 太近的元素（< 0.3" 间隙）或几乎接触的卡片/部分
- 不均匀的间隙（一处大空白区域，另一处拥挤）
- 距离幻灯片边缘的边距不足（< 0.5"）
- 列或类似元素不一致对齐
- 低对比度文本（例如，米色背景上的浅灰色文本）
- 低对比度图标（例如，深色背景上没有对比圆圈的深色图标）
- 太窄导致过度换行的文本框
- 遗留的占位符内容

对于每张幻灯片，列出问题或关注区域，即使是轻微的。

阅读并分析这些图片：
1. /path/to/slide-01.jpg（预期：[简要描述]）
2. /path/to/slide-02.jpg（预期：[简要描述]）

报告发现的所有问题，包括轻微的。
```

### 验证循环

1. 生成幻灯片 → 转换为图片 → 检查
2. **列出发现的问题**（如果没有找到，更批判性地再看一遍）
3. 修复问题
4. **重新验证受影响的幻灯片**——一个修复经常会产生另一个问题
5. 重复直到完整通过没有发现新问题

**在完成至少一次修复和验证循环之前不要声明成功。**

## 转换为图片

将演示文稿转换为单独的幻灯片图片以进行直观检查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

这会创建 `slide-01.jpg`、`slide-02.jpg` 等。

要在修复后重新渲染特定幻灯片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```
