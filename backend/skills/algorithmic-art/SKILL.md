算法理念是通过代码表达的计算美学运动。输出.md文件（理念）、.html文件（交互式查看器）和.js文件（生成算法）。

这分为两个步骤：
1. 算法理念创建（.md文件）
2. 通过创建p5.js生成艺术来表达（.html + .js文件）

首先，执行此任务：

## 算法理念创建

首先，创建一个算法理念（不是静态图像或模板），将通过以下方式解释：
- 计算过程、涌现行为、数学之美
- 种子随机性、噪声场、有机系统
- 粒子、流场、力场
- 参数变化和受控混沌

### 关键理解
- 接收什么：用户的一些细微输入或指令需要考虑，但作为基础使用；不应限制创作自由。
- 创建什么：一个算法理念/生成美学运动。
- 接下来发生什么：同一版本接收理念并通过代码表达它 - 创建p5.js草图，90%是算法生成，10%是基本参数。

考虑这种方法：
- 为生成艺术运动撰写宣言
- 下一阶段涉及编写使其生动的算法

理念必须强调：算法表达。涌现行为。计算之美。种子变化。

### 如何生成算法理念

**命名运动**（1-2个词）："有机湍流" / "量子和谐" / "涌现静谧"

**阐述理念**（4-6段 - 简洁但完整）：

要捕捉算法本质，表达这个理念如何通过以下方式体现：
- 计算过程和数学关系？
- 噪声函数和随机性模式？
- 粒子行为和场动力学？
- 时间演变和系统状态？
- 参数变化和涌现复杂性？

**关键指南：**
- **避免冗余**：每个算法方面应只提及一次。避免重复关于噪声理论、粒子动力学或数学原理的概念，除非增加新的深度。
- **反复强调工艺**：理念必须多次强调最终算法应看起来像是经过无数小时开发的，经过精心优化，并来自该领域绝对顶尖的人。这种框架至关重要 - 重复使用"精心制作的算法"、"深度计算专业知识的产品"、"痛苦优化"、"大师级实现"等短语。
- **留出创作空间**：具体说明算法方向，但要简洁，以便下一个Claude有足够的空间在极高水平的工艺上做出解释性实现。

理念必须指导下一个版本通过算法表达想法，而不是通过静态图像。美存在于过程中，而不是最终帧中。

### 理念示例

**"有机湍流"**
理念：受自然法则约束的混沌，从无序中涌现的秩序。
算法表达：由分层Perlin噪声驱动的流场。数千个粒子跟随矢量力，它们的轨迹累积成有机密度图。多个噪声八度创建湍流区域和宁静区域。颜色基于速度和密度 - 快粒子燃烧明亮，慢粒子褪色成阴影。算法运行直到平衡 - 通过无数次迭代精心调整的平衡，每个参数都经过精心优化。

**"量子和谐"**
理念：表现出波状干涉模式的离散实体。
算法表达：在网格上初始化的粒子，每个粒子携带一个通过正弦波演化的相位值。当粒子靠近时，它们的相位会干涉 - 建设性干涉创建亮节点，破坏性创建空洞。简谐运动生成复杂的涌现曼陀罗。精心频率校准的结果，每个比率都经过精心选择以产生共振美。

**"递归低语"**
理念：跨尺度的自相似性，在有限空间中的无限深度。
算法表达：递归细分的分支结构。每个分支略微随机化但受黄金比例约束。L系统或递归细分生成既数学又有机的树状形式。微妙的噪声扰动打破完美对称。线宽随每个递归级别递减。每个分支角度都是深度数学探索的产物。

**"场动力学"**
理念：通过其对物质的影响使不可见的力可见。
算法表达：由数学函数或噪声构建的矢量场。粒子在边缘诞生，沿着场线流动，当达到平衡或边界时死亡。多个场可以吸引、排斥或旋转粒子。可视化只显示轨迹 - 不可见力的幽灵般证据。通过力平衡精心编排的计算舞蹈。

**"随机结晶"**
理念：随机过程结晶成有序结构。
算法表达：随机圆形包装或Voronoi镶嵌。从随机点开始，通过松弛算法演化。细胞相互推开直到平衡。基于细胞大小、邻居数量或到中心距离的颜色。涌现的有机镶嵌既随机又必然。每个种子产生独特的结晶美 - 大师级生成算法的标志。

*这些是压缩示例。实际算法理念应为4-6个实质性段落。*

### 基本原则
- **算法理念**：创建通过代码表达的计算世界观
- **过程胜于产品**：始终强调美从算法执行中涌现 - 每次运行都是独特的
- **参数表达**：想法通过数学关系、力、行为传达 - 不是静态构图
- **艺术自由**：下一个Claude以算法方式解释理念 - 提供创作实现空间
- **纯生成艺术**：这是关于制作活算法，不是带有随机性的静态图像
- **专家工艺**：反复强调最终算法必须感觉精心制作，通过无数次迭代优化，由计算美学领域绝对顶尖的人深度专业知识的产品

**算法理念应为4-6段长。** 用诗意的计算理念填充实它，将预期愿景结合在一起。避免重复相同观点。将此算法理念输出为.md文件。

**关键步骤**：在实现算法之前，识别原始请求中的微妙概念线索。

**基本原则**：
概念是**嵌入在算法本身中的微妙、小众引用** - 不总是字面的，总是复杂的。熟悉主题的人应该能直观感受到，而其他人只是体验大师级的生成构图。算法理念提供计算语言。推断的概念提供灵魂 - 静静地编织到参数、行为和涌现模式中的概念DNA。

这**非常重要**：引用必须如此精致，以至于它增强了作品的深度而不会宣布自己。像爵士音乐家通过算法和谐引用另一首歌一样思考 - 只有知道的人才会捕捉到，但每个人都会欣赏生成之美。

### 技术要求

**种子随机性（Art Blocks模式）**：
```javascript
// 始终使用种子以确保可重现性
let seed = 12345; // 或从用户输入哈希
randomSeed(seed);
noiseSeed(seed);
```

**参数结构 - 遵循理念**：

要建立从算法理念中自然出现的参数，考虑："这个系统的哪些品质可以调整？"

```javascript
let params = {
  seed: 12345,  // 始终包含种子以确保可重现性
  // 颜色
  // 添加控制您的算法的参数：
  // - 数量（多少？）
  // - 尺度（多大？多快？）
  // - 概率（多可能？）
  // - 比率（什么比例？）
  // - 角度（什么方向？）
  // - 阈值（行为何时改变？）
};
```

**为了设计有效的参数，专注于系统需要可调的属性，而不是考虑"模式类型"。**

**核心算法 - 表达理念**：

**关键**：算法理念应决定构建什么。

要通过代码表达理念，避免思考"应该使用哪种模式？"而应思考"如何通过代码表达这个理念？"

如果理念是关于**有机涌现**，考虑使用：
- 随时间累积或生长的元素
- 受自然规则约束的随机过程
- 反馈循环和交互

如果理念是关于**数学之美**，考虑使用：
- 几何关系和比率
- 三角函数和和谐
- 精确计算创建意外模式

如果理念是关于**受控混沌**，考虑使用：
- 严格边界内的随机变化
- 分岔和相变
- 从无序中涌现的秩序

**算法从理念中流出，而不是从选项菜单中流出。**

要指导实现，让概念本质指导创造性和原创性的选择。构建一些表达此特定请求愿景的东西。

**画布设置**：标准p5.js结构：
```javascript
function setup() {
  createCanvas(1200, 1200);
  // 初始化您的系统
}

function draw() {
  // 您的生成算法
  // 可以是静态的（noLoop）或动画的
}
```

### 工艺要求

**关键**：要达到精通，创建感觉像是由大师生成艺术家通过无数次迭代产生的算法。仔细调整每个参数。确保每个模式都有目的地涌现。这不是随机噪声 - 这是通过深度专业知识优化的受控混沌。

- **平衡**：复杂性但无视觉噪声，秩序但无刚性
- **色彩和谐**：深思熟虑的调色板，不是随机RGB值
- **构图**：即使在随机性中，也要保持视觉层次和流程
- **性能**：平滑执行，如果动画则优化实时性能
- **可重现性**：相同种子始终产生相同输出

### 输出格式

输出：
1. **算法理念** - 作为markdown或文本解释生成美学
2. **单个HTML工件** - 基于`templates/viewer.html`构建的自包含交互式生成艺术（见步骤0和下一节）

HTML工件包含所有内容：p5.js（来自CDN）、算法、参数控制和UI - 所有内容都在一个文件中，可立即在claude.ai工件或任何浏览器中工作。从模板文件开始，而不是从头开始。

## 交互式工件创建

**提醒：`templates/viewer.html`应该已经读取（见步骤0）。使用该文件作为起点。**

要允许探索生成艺术，创建一个自包含的HTML工件。确保此工件在claude.ai或任何浏览器中立即工作 - 无需设置。内联嵌入所有内容。

### 关键：什么是固定的vs可变的

`templates/viewer.html`文件是基础。它包含所需的确切结构和样式。

**固定（始终完全如所示包含）：**
- 布局结构（标题、侧边栏、主画布区域）
- Anthropic品牌（UI颜色、字体、渐变）
- 侧边栏中的种子部分：
  - 种子显示
  - 上一个/下一个按钮
  - 随机按钮
  - 跳转到种子输入 + Go按钮
- 侧边栏中的操作部分：
  - 重新生成按钮
  - 重置按钮

**可变（为每件艺术品定制）：**
- 整个p5.js算法（setup/draw/类）
- 参数对象（定义艺术需要什么）
- 侧边栏中的参数部分：
  - 参数控制的数量
  - 参数名称
  - 滑块的最小/最大/步长值
  - 控制类型（滑块、输入等）
- 颜色部分（可选）：
  - 一些艺术需要颜色选择器
  - 一些艺术可能使用固定颜色
  - 一些艺术可能是单色的（不需要颜色控制）
  - 根据艺术需求决定

**每件艺术品都应该有独特的参数和算法！** 固定部分提供一致的UX - 其他一切都表达独特愿景。

### 必需功能

**1. 参数控制**
- 数字参数的滑块（粒子数量、噪声尺度、速度等）
- 调色板颜色的颜色选择器
- 参数更改时的实时更新
- 恢复默认值的重置按钮

**2. 种子导航**
- 显示当前种子号
- "上一个"和"下一个"按钮循环种子
- "随机"按钮用于随机种子
- 跳转到特定种子的输入字段
- 请求时生成100个变体（种子1-100）

**3. 单个工件结构**
```html
<!DOCTYPE html>
<html>
<head>
  <!-- p5.js from CDN - 始终可用 -->
  <script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.7.0/p5.min.js"></script>
  <style>
    /* 所有样式内联 - 简洁、最小 */
    /* 画布在顶部，控制在下方 */
  </style>
</head>
<body>
  <div id="canvas-container"></div>
  <div id="controls">
    <!-- 所有参数控制 -->
  </div>
  <script>
    // 所有p5.js代码内联在此处
    // 参数对象、类、函数
    // setup() 和 draw()
    // UI处理程序
    // 所有内容自包含
  </script>
</body>
</html>
```

**关键**：这是一个单一工件。无外部文件，无导入（除了p5.js CDN）。所有内容内联。

**4. 实现细节 - 构建侧边栏**

侧边栏结构：

**1. 种子（固定）** - 始终完全如所示包含：
- 种子显示
- 上一个/下一个/随机/跳转按钮

**2. 参数（可变）** - 为艺术创建控制：
```html
<div class="control-group">
    <label>参数名称</label>
    <input type="range" id="param" min="..." max="..." step="..." value="..." oninput="updateParam('param', this.value)">
    <span class="value-display" id="param-value">...</span>
</div>
```
根据参数数量添加尽可能多的control-group div。

**3. 颜色（可选/可变）** - 如果艺术需要可调颜色则包含：
- 如果用户应控制调色板则添加颜色选择器
- 如果艺术使用固定颜色则跳过此部分
- 如果艺术是单色的则跳过
- 根据艺术需求决定

**4. 操作（固定）** - 始终完全如所示包含：
- 重新生成按钮
- 重置按钮
- 下载PNG按钮

**要求**：
- 种子控制必须工作（上一个/下一个/随机/跳转/显示）
- 所有参数必须有UI控制
- 重新生成、重置、下载按钮必须工作
- 保持Anthropic品牌（UI样式，不是艺术颜色）

### 使用工件

HTML工件立即工作：
1. **在claude.ai中**：显示为交互式工件 - 立即运行
2. **作为文件**：保存并在任何浏览器中打开 - 无需服务器
3. **分享**：发送HTML文件 - 完全自包含

## 创作过程

**用户请求** → **算法理念** → **实现**

每个请求都是独特的。过程包括：

1. **解释用户意图** - 寻求什么美学？
2. **创建算法理念**（4-6段）描述计算方法
3. **在代码中实现** - 构建表达此理念的算法
4. **设计适当参数** - 什么应该是可调的？
5. **构建匹配的UI控制** - 这些参数的滑块/输入

**常量**：
- Anthropic品牌（颜色、字体、布局）
- 种子导航（始终存在）
- 自包含HTML工件

**其他都是可变的**：
- 算法本身
- 参数
- UI控制
- 视觉结果

要获得最佳结果，请相信创造力，让理念指导实现。
